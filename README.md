# AI Data Analyst Agent

**Ask business questions in plain English. Get executive-ready analysis from your Snowflake data — in seconds.**

No SQL knowledge required. No dashboards to maintain. No waiting for analysts. Just ask.

---

## Why This Exists

Business teams wait days for data answers. Analysts are overwhelmed. Dashboards go stale. Self-service BI tools require training nobody has time for.

This agent changes that: a single API call turns a plain-English question into a structured executive report — complete with the SQL it ran, the tables it queried, and the findings it discovered.

---

## 💡 Business Value

| Problem | This Agent Solves It |
|---|---|
| "We need an analyst to pull that data" | Anyone asks in plain English, gets an answer in seconds |
| SQL expertise required for every query | GPT-4o writes, validates, and executes the SQL |
| Stale dashboards miss new questions | Ad-hoc analysis on live Snowflake data, any time |
| Reports take hours to write | 5-section executive markdown report, auto-generated |
| Risk of accidental data modification | SQL validator enforces SELECT-only, blocks DDL/DML |

**Who benefits:** Business analysts, product managers, executives, sales ops, finance teams — anyone who needs data answers without writing code.

---

## Key Capabilities

- **Natural language to SQL** — GPT-4o generates optimized, CTE-based Snowflake SQL from plain-English questions
- **Automatic schema discovery** — the agent finds relevant tables itself; no manual configuration per query
- **Safety-first SQL execution** — code-level validator blocks any non-SELECT statement before it reaches Snowflake
- **9-node reasoning pipeline** — structured LangGraph workflow: intent extraction → planning → discovery → selection → SQL generation → validation → execution → analysis → reporting
- **Executive reports** — every response is a 5-section markdown report: summary, findings, anomalies, key metrics, recommendations
- **Retry resilience** — exponential backoff on all LLM and database calls via tenacity
- **Structured logging** — JSON logs via structlog for production observability
- **OpenAPI docs** — Swagger UI auto-generated by FastAPI at `/docs`

## LangGraph Pipeline (9 Nodes)

```
 ┌─────────────────────────────────────────────────────────────────┐
 │                    LangGraph StateGraph                         │
 │                                                                 │
 │  [1] intent_node          GPT-4o extracts intent, metrics,     │
 │          │                dimensions, time period               │
 │          ▼                                                      │
 │  [2] planner_node         GPT-4o creates 4–7 step plan         │
 │          │                                                      │
 │          ▼                                                      │
 │  [3] metadata_discovery_node                                    │
 │          │                FastMCP → INFORMATION_SCHEMA          │
 │          ▼                                                      │
 │  [4] dataset_selector_node                                      │
 │          │                GPT-4o selects ≤5 relevant tables     │
 │          ▼                                                      │
 │  [5] sql_generator_node   GPT-4o generates CTE-based SQL       │
 │          │                                                      │
 │          ▼                                                      │
 │  [6] sql_validator_node   Code check: SELECT-only, no DDL/DML  │
 │          │                                                      │
 │    ┌─────┴──────┐                                               │
 │   PASS        FAIL                                              │
 │    │            └──────────────────────┐                        │
 │    ▼                                   │                        │
 │  [7] sql_executor_node                 │                        │
 │    FastMCP → Snowflake                 │                        │
 │    rows + columns returned             │                        │
 │          │                             │                        │
 │          ▼                             ▼                        │
 │  [8] analyst_node         GPT-4o: findings, anomalies,         │
 │          │                key metrics (error-aware)             │
 │          ▼                                                      │
 │  [9] response_node        GPT-4o: 5-section executive report   │
 └─────────────────────────────────────────────────────────────────┘
```

## 🚀 Quick Start

**Prerequisites:** Python 3.10+, a Snowflake account, an OpenAI API key.

### 1. Clone the repository

```bash
git clone https://github.com/srimukund289/lanngraph_sf.git
cd lanngraph_sf
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

```bash
cp .env.example .env
# Edit .env with your credentials
```

### 4. Start the server

```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

### 5. Ask your first question

```bash
curl -X POST http://localhost:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the total orders in 1996?"}'
```

API docs available at: http://localhost:8000/docs

## Configuration

Set these environment variables in `.env` or your deployment environment.

| Variable | Required | Description |
|---|---|---|
| `OPENAI_API_KEY` | Yes | OpenAI API key for GPT-4o |
| `SNOWFLAKE_ACCOUNT` | Yes | Snowflake account identifier (e.g. `xy12345.us-east-1`) |
| `SNOWFLAKE_USER` | Yes | Snowflake username |
| `SNOWFLAKE_PASSWORD` | Yes | Snowflake password |
| `SNOWFLAKE_WAREHOUSE` | Yes | Compute warehouse name |
| `SNOWFLAKE_DATABASE` | Yes | Default database |
| `SNOWFLAKE_SCHEMA` | Yes | Default schema |
| `SNOWFLAKE_ROLE` | No | Snowflake role (uses account default if omitted) |

Example `.env`:

```env
OPENAI_API_KEY=sk-...
SNOWFLAKE_ACCOUNT=xy12345.us-east-1
SNOWFLAKE_USER=analyst
SNOWFLAKE_PASSWORD=yourpassword
SNOWFLAKE_WAREHOUSE=COMPUTE_WH
SNOWFLAKE_DATABASE=SALES_DB
SNOWFLAKE_SCHEMA=PUBLIC
SNOWFLAKE_ROLE=ANALYST_ROLE
```

## FastMCP Tools

The `snowflake_mcp_server.py` exposes six tools to the LangGraph pipeline:

| Tool | Description |
|---|---|
| `execute_query(sql, max_rows=1000)` | Run SELECT queries; LIMIT enforced automatically |
| `list_databases()` | List all accessible Snowflake databases |
| `list_schemas(database)` | List schemas in a database |
| `list_tables(database, schema)` | List tables in a schema |
| `describe_table(database, schema, table)` | Get column names, types, and metadata |
| `discover_metadata(database, schema)` | Full nested catalog: `{db: {schema: {table: [cols]}}}` |

All tools run in-process — no remote HTTP server required.

---

## Safety

- **SELECT-only enforcement** — `sql_validator_node` rejects any query containing DDL (`CREATE`, `DROP`, `ALTER`) or DML (`INSERT`, `UPDATE`, `DELETE`) before it reaches Snowflake.
- **CTE-aware table whitelist** — the validator checks that all referenced tables were discovered in the metadata step.
- **Row limits** — `execute_query` enforces a configurable `max_rows` cap (default 1000) to prevent runaway queries.
- **No query rewriting at execution time** — the SQL that was validated is the SQL that runs.
